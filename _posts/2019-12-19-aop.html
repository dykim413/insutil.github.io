--- layout: post title: "AOP" subtitle: "Aspect Oriented Programming" date: 2019-12-19 category: [study] author: dykim413 comments: true --- Aspect Oriented Programming 에 대해 아래의 순서대로 알아봅니다. ● AOP 정의 ● AOP 개발 방법 AOP는 모듈화된 부가기능(Advice) 과 적용 대상(Pointcut) 의 조합 (Advisor) 을 통해 여러 오브젝트에 중복되고 산재해 있는 비슷한 성격의 기능을 쉽게 모듈화 하여 적용하고 관리할 수 있게 해주는 기술을 말합니다. 스프링에서 AOP 의 개발 방법은 4가지로 정리할 수 있다. ● AOP 인터페이스 구현 과 Bean 등록을 통한 방법 ● AOP 인터페이스 구현과 AOP 네임스페이스의 태그를 이용하는 방법 ● 임의의 자바 클래스와 AOP 네임스페이스의 aop:aspect 를 이용하는 방법 ● @AspectJ 어노테이션을 이용하는 방법 전통적인 AOP 개발 방법은 스프링이 제공하는 AOP 관련 인터페이스를 직접 구현하는 방식으로 이루어 진다. AOP 구성 요소를 모두 Class 로 개발하고 이를 빈으로 등록한다. 이후 Advice 와 PointCut 을 조합하여 어드바이저를 구성하고 이를 Proxy로 만들어 빈에게 적용해 주는 자동 프록시 생성기 빈을 등록하면 된다. 표현이 깔끔하지 못하고 괜히 복잡하고 어렵게 느껴진다. 직접 AOP 관련 인터페이스를 구현하여 이를 빈으로 등록하고 다시 Proxy로 만들고 하는등의 방법은 잘 사용되지 않는 추세이다. 따라서 우리는 @AspectJ 어노테이션을 이용한 방법을 알아 볼것이다. 여기서 한가지 짚고 넘어가야 할것은 @AspectJ 를 AspectJ 혼동하지 말라는 것이다. @Aspect 도 여타 방법과 비슷하게 스프링의 프록시 기반 AOP를 만들때 사용하는 방법의 하나일 뿐이다.
